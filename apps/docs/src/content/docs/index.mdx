## Зачем Pane

<CardGrid>
  <Card title="Настройка устройства через USB" icon="seti:apple">
    Подключи телефон, нажми Add device — Pane через USB поставит root
    CA (на iOS и рутованном Android — полный авто; на non-root
    Android из-за блокировки Google нужен один ручной шаг по
    инструкции в UI), пробросит трафик через `adb reverse` и
    выставит PAC-прокси. Wi-Fi настраивать не надо.
  </Card>
  <Card title="MITM HTTPS с расшифровкой" icon="seti:lock">
    Per-host leaf-сертификаты, выписанные локальным root CA. Каждый
    запрос приложения попадает в список как готовый capture — заголовки,
    тело, тайминги — без дополнительной настройки на устройстве.
  </Card>
  <Card title="Подмена или патчинг ответов" icon="random">
    Можно заменить ответ целиком, а можно прокинуть запрос к серверу и
    подменить только одно поле (`user.uid`, токен, timestamp). В Patch-
    режиме сервер-генерируемые данные остаются настоящими.
  </Card>
  <Card title="Удобная фильтрация" icon="magnifier">
    Маленький DSL в строке поиска: `host:api.example.com`, `status:5..`,
    `!error:tls_handshake` плюс barewords. Сохранённые фильтры живут
    в боковой панели.
  </Card>
  <Card title="Двуязычный интерфейс" icon="translate">
    English по умолчанию + полная русская локализация всех экранов
    (Captures, Rules, Devices, Settings, manual-install guide). Переключение
    в Settings → Appearance → Language, реактивно, без перезапуска.
  </Card>
</CardGrid>

## Быстрый пример

```text
# Фильтр в списке captures
host:rc1.test.dev-og.com status:200..299
!error:pinning method:POST
```

```text
# Stub — заменить ответ целиком
Match:    POST /api/auth, params login=root
Mode:     Stub
Status:   200
Body:     {"token": "test-stub-token", "user": {"fio": "Тестовый"}}
```

```text
# Patch — оставить настоящий токен, подменить одно поле
Match:    POST /api/auth, params login=root
Mode:     Patch
Patches:  set user.fio = "Тестовый"
          set user.uid = 777
```
