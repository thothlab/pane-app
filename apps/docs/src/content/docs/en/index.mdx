## Why Pane

<CardGrid>
  <Card title="Trust setup over USB" icon="seti:apple">
    Plug a phone in, hit Add device. Pane installs the root CA over USB
    (fully automatic on iOS and rooted Android; one manual step on
    non-root Android because Google blocks programmatic CA install,
    Pane walks you through it in the UI), tunnels traffic via
    `adb reverse`, and sets a PAC-based proxy. No Wi-Fi setup needed.
  </Card>
  <Card title="MITM HTTPS, decoded" icon="seti:lock">
    Per-host leaf certificates minted from a local root CA. Every request
    your app makes shows up as a parsed capture — headers, body, timing —
    with no extra config on the phone side.
  </Card>
  <Card title="Stub or patch responses" icon="random">
    Replace a whole response with a canned one, or forward to the real
    server and patch a single field (`user.uid`, a token, a timestamp).
    Patch mode keeps server-generated state fresh.
  </Card>
  <Card title="Filtering that scales" icon="magnifier">
    A small DSL on the search bar: `host:api.example.com`, `status:5..`,
    `!error:tls_handshake`, plus barewords. Saved filters live in the
    sidebar.
  </Card>
</CardGrid>

## Quick taste

```text
# Filter the capture list
host:rc1.test.dev-og.com status:200..299
!error:pinning method:POST
```

```text
# Stub a response — replace the whole body
Match:    POST /api/auth, params login=root
Mode:     Stub
Status:   200
Body:     {"token": "test-stub-token", "user": {"fio": "Тестовый"}}
```

```text
# Patch a response — keep the real token, swap one field
Match:    POST /api/auth, params login=root
Mode:     Patch
Patches:  set user.fio = "Тестовый"
          set user.uid = 777
```
